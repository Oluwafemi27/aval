# M4 Conformance Fixture Provenance

The two valid `.avl` files in the parent directory are deterministic output
from `packages/format/test/fixture-generator.ts` and the M4 canonical writer.
Their SHA-256 values cover the whole checked-in file and exist only to make an
accidental fixture change visible during review. M4 does not verify embedded
media digests at runtime.

| --- | ---: | --- | --- |
| `reference-loop.avl` | 1280 bytes | `d297074a31c66652c3051a69617f899085ee26f8fdfea5254cb9a4cad679b339` | One state and a three-frame `reference-rgba-v0` loop. |
| `reference-graph.avl` | 4992 bytes | `1ffbe4f530b2955a65635c216c5dbfdceba02911ea86763c3dda31266d397ced` | Loop, finite, and held bodies plus every transition start. |

Regenerate the binary fixtures from the repository root with:

```sh
AVL_UPDATE_CONFORMANCE_FIXTURES=1 npx vitest run packages/format/test/conformance.test.ts
```

The conformance test requires regenerated bytes to match the generator and
requires this document to contain their current size and digest. Review and
update this table intentionally whenever the frozen 0.1 bytes change.

## Malformed corpus

Malformed inputs are generated in memory from named, bounded mutations; this
directory intentionally does not contain hundreds of nearly identical binary
files. `conformance.test.ts` covers:

- duplicate escaped and dangerous JSON keys;
- unsafe `uint64` values and extreme sample counts;
- false index lengths, record ordering, zero/huge sample sizes, and false key
  flags;
- nonzero padding, gaps, overlaps, aliases, and trailing bytes;
- profile mismatches and malformed reference-frame headers.

`mutation-fuzz.test.ts` adds fixed-seed single-byte, slice, insertion, deletion,
and structured header/index/padding/profile mutations. Every mutation must
either return a recursively frozen valid layout or throw `FormatError`; a
built-in exception is always a failure.

Complete-asset validation does not claim H.264 decodability, digest integrity,
network integrity, or successful media decode.
