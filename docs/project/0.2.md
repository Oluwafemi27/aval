# Compiler project 0.2

Project `0.2` is strict JSON describing author-sized sources and an explicit
canvas and frame rate,
author-defined states, units, ports, edges, bindings, readiness,
renditions, alpha policy, and output limits. Unknown fields and ambiguous
timelines are rejected.

The compiler resolves source files relative to the project, never from asset
URLs. Reproducible builds require the complete recorded Node, compiler,
FFmpeg/FFprobe executable, configuration, encoder, and calibration fingerprint.
Another supported tool build may be semantically conformant without producing
the same AVC bytes.

Canvas/rendition dimensions and half-open unit ranges are authoritative author
input. The compiler does not choose a smaller size or shorter range. See the
[authoring guide](../compiler/authoring-video-and-states.md).

Source pixels are equally authoritative. Loop and edge boundaries are analyzed
and recorded as `pass`, `review`, or declared `cut` in compiler build details.
`review` is a non-fatal quality warning: the compiler preserves the selected
frames and does not synthesize a blend.

Rendition encoding in `0.2` is the legacy fixed policy: libx264 `medium` with
average and peak bitrate fields. Use [project 0.3](0.3.md) when authors need
capped CRF or another allowlisted preset.
