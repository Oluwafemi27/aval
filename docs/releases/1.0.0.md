# AVAL 1.0.0

Status: release candidate engineering implemented; publication and named-profile
certification remain protected external gates.

The release synchronizes `@pixel-point/aval-graph`, `@pixel-point/aval-format`, `@pixel-point/aval-compiler`,
`@pixel-point/aval-player-web`, and `@pixel-point/aval-element` at `1.0.0`. It keeps wire format `0.1`
and compiler project `0.3` (with legacy `0.1` and `0.2` inputs retained).

Highlights include user-defined state graphs, continuous decoded partial loops,
resident reversal, packed alpha, host-owned fallback markup, bounded range and
integrity loading, page-wide resource arbitration, and an SSR-safe custom
element.

Known boundaries:

- web desktop only; no native/mobile release;
- exact H.264/WebCodecs/WebGL behavior is capability-probed;
- unsupported animation leaves optional host fallback markup visible;
- runtime scheduling reports do not prove physical display scan-out;
- observed-display evidence is not measured until a separate report passes;
- FFmpeg/libx264 and codec/content licensing remain external obligations.
